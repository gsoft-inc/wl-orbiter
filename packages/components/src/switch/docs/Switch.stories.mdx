## Guidelines

### When to use

- To trigger an immediate action such as an activation/deactivation.

### Content

- Have a short and descriptive label.
- Be self explanatory without relying on an icon. Using an icon to carry information will render it non-accessible.

### Accessibility

- Do not change the label based on the state of the switch. The label should represent the "on/checked" state of the switch.

### Checkbox, Radio or Switch

- If the objective is to select multiples elements from a list, the [checkbox](?path=/docs/checkbox--default-story) should be used instead.
- If the objective is to select one value out of a set of elements the [radio](?path=/docs/radio--default-story) should be used instead.

A switch can be checked.

<Preview>

        <Switch defaultChecked>Engines</Switch>
    </Story>
</Preview>

### No label

A switch can be rendered without a label.

<Preview>
    <Story name="no label">
        <Switch aria-label="Engines" />
    </Story>
</Preview>

### Disabled

A switch can be disabled.

<Preview>
    <Story name="disabled">
        <Switch disabled>Engines</Switch>
    </Story>
</Preview>

### Icon

A switch can have [icons](?path=/docs/icon-gallery--page) after his text.

<Preview>
    <Story name="icon">
        <Switch>
            <Text>Pressurization</Text>
            <SparklesIcon />
        </Switch>
    </Story>
</Preview>

### Counter

A switch can have a [counter](?path=/docs/counter--default-story) after his text.

<Preview>
    <Story name="counter">
        <Switch>
            <Text>Engines</Text>
            <Counter>6</Counter>
        </Switch>
    </Story>
</Preview>

### Size

A switch can vary in size.

    <Story name="size">

    </Story>

### Reverse

A checkbox checkmark can be to the left or to the right of his label.

<Preview>

            <Counter variant="divider">6</Counter>
        </Switch>
    </Story>
</Preview>
